## design document 
0. overview (abstract)
A. definition
    1. situation (context, problem statement)
    2. goals (& non-goals)
        - success criteria
    3. people
        - audience / user journeys
        - contributors (work collaborators)
        - stakeholders & roles
        - reviewers
B. approach
    6. proposed solution (design guidelines, technical architecture)
        - components (specs)
        - existing solution workflow
        - proposed workflow
        - alternatives considered
    7. risks & resources
        - constraints & assumptions, dependencies
    8. methodology & methods
        - ethos
        - making tools
        - starting small
B. objectives
    4. deliverables 
    5. outcomes & impacts (benefit, recommendation)
D. details
    9. steps (scope & implementation)
        - pseudocode
        - user journeys?
        - how to's, user testing
        - testing, monitoring, auditing, alerting
        - communication plan
    10. scope 
        - timelines (milestones & checkpoints)
        - budget
11. discussion & insights (+ conclusions, execution, message/takeaway)
appendices: glossary, style guide, visual examples, code examples, updates
